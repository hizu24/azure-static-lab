通信経路振り分けの練習

このページは、Prisma Access、プロキシ、直接通信、社内システム向け通信の 経路分けを理解するための練習です。

経路ルール

  • microsoft.com / office.com / teams.microsoft.com → 直接通信
  • intra.company.local / internal.company.local → 社内システム向け
  • proxy-only.example.com → プロキシ経由
  • それ以外 → Prisma Access経由

案件に置き換えると

  • 直接通信：Prismaを通さず、端末からインターネットへ出す通信
  • Prisma Access経由：GlobalProtect経由でPrismaに流す通信
  • プロキシ経由：既存プロキシや新規プロキシを通す通信
  • 社内システム向け：社内ネットワークや閉域向けの通信
